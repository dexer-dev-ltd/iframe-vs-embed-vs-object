embed or object or iframe

Tag သုံးမျိုးလုံး ကို HTML5 မာ သုံးလို့ရတယ်။ သုံးမျိုးလုံးက အသုံးအနုန်း အရ ရူပ်ထွေးနိုင်လောက်အောင်ကို ဆင်တူကြတယ်။

<embed/>
  • အဓိကတော့ non-HTML application နဲ့ လုပ်ရှားနေတဲ့ content (Animations) တွေမာ အသုံးများပါတယ်။
  • <embed> က HTML5 မတိုင်ခင် Standard မဖြစ်ခဲ့ဘူး HTML5 မာ ပြန်ပြီး Standardised ထားတော့ ခောတ်သစ် technology ဖြစ်သွားပြန်တယ်။
  • <svg> နဲ့ အလုပ်လုပ်တဲ့နေအခါ getSVGDocument() method သုံးပြီး embedded SVG document ကို Parent Element ကနေ access လုပ်လို့ရပါတယ်။
  • <embed> ရဲ့ အဓိက ပြဿနာကတော့ browser support ပါပဲ။ ဒီမာ စစ်ပါ
<object></object>
  • Animation တွေကို support ပေးတဲ့အတွက် svg တွေအတွက် အကောင်းဆုံးကတော့ <object> ပါပဲ။
  • <object> မှာ ကိုယ်သုံးထားတဲ့ plugin မရှိရင် မရှိကြောင်း fallback content ထည့်ပေးလို့ရတယ်။
  • <iframe> အစား <object> ကို သုံးသင့်တဲ့ အကြောင်းရင်းတခုက - <object> ဟာ သူ့ထဲ ထည့်ထားတဲ့ content ကို <object> ရဲ့ dimension နဲ့ fit အောင်ထားနိုင်တယ်။
<iframe></iframe>
  • အဓိကတော့ အခြား domain, sub-domain ဒါမှမဟုတ် ကိုယ့် domain ထဲက external resources တွေကို ထည့်တဲ့ချိန် သုံးတယ်။
  • သူ့အားသားချက်က ကိုယ် set လိုက်တဲ့ content က live ဖြစ်တယ်။

My Point of View

object ကို ရှေးခောတ် <embed> support မလုပ်တဲ့ browser မာပဲသုံးပါ။

Browser က <embed> support ရင်တော့ အကောင်းဆုံးတော့ embed ကိုပဲသုံးပါ။

ကိုယ်ချိတ်ချင်တဲ့ Content က Static ဖြစ်ပြီးတော့ SVG, or အခြား Flash Content ဖြစ်နေရင် တော့ <embed> ဒါမှမဟုတ် <object> ကိုပဲ သုံးပါ

Content က Live Content ဒါမှမဟုတ် HTML Content တွေဆိုရင်တော့ <iframe> က အကောင်းဆုံးပါပဲ။
